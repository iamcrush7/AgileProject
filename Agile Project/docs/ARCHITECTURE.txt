  └─────────────────────────┘

  └─────────────────────────┘

## Request/Response Flow

```
┌──────────────────────────────────────────────────────────────┐
│                   AJAX REQUEST FLOW                          │
└──────────────────────────────────────────────────────────────┘

[Browser]
    │
    ├─ User submits form
    │
    └─→ JavaScript intercepts (e.preventDefault())
         │
         ├─ Create FormData
         │
         └─→ fetch('handler.php', {
                    method: 'POST',
                    body: formData
                })
             │
             └─→ [Server]
                 │
                 ├─ Handler processes data
                 │
                 └─→ echo json_encode({
                         'success': true/false,
                         'message': 'text',
                         'redirect': 'page.php'
                     })
                 │
                 └─→ [Browser]
                     │
                     ├─ Parse JSON
                     │
                     ├─ Show message
                     │
                     └─→ Redirect if success
                         │
                         └─→ [New Page]
```

## Security Layers

```
┌────────────────────────────────────────────────────────────┐
│                   SECURITY IMPLEMENTED                      │
└────────────────────────────────────────────────────────────┘

Layer 1: Client-Side Validation
├─ HTML5 required attribute
├─ Email format check
├─ Password length check
└─ Confirm password match

Layer 2: Server-Side Validation
├─ Check all fields provided
├─ Validate email format
├─ Check password length
├─ Verify password match
└─ Check duplicate email

Layer 3: Database Security
├─ Prepared Statements (prevent SQL injection)
├─ Parameterized queries
├─ Use bind_param()
└─ Never concatenate user input

Layer 4: Password Security
├─ Hash with password_hash()
├─ Verify with password_verify()
├─ Never store plain text
└─ Use PASSWORD_DEFAULT algorithm

Layer 5: Session Security
├─ Session-based authentication
├─ User ID stored in session
├─ Protected pages check session
└─ Admin pages check user type

Layer 6: Error Handling
├─ Don't expose database errors to users
├─ Log errors server-side
├─ Show generic messages to users
└─ Return JSON for AJAX
```

## Database Connection

```
┌────────────────────────────────────────────────────────┐
│              CONNECTION ARCHITECTURE                     │
└────────────────────────────────────────────────────────┘

[config.php]
    │
    ├─ Server: localhost
    ├─ Username: root
    ├─ Password: (empty)
    └─ Database: agile
         │
         └─ mysqli_connect()
             │
             └─ $conn object
                 │
                 ├─ Used by login_handler.php
                 ├─ Used by register_handler.php
                 ├─ Used by all pages
                 └─ prepare() & execute()
                     │
                     └─ Bind parameters
                         │
                         └─ Execute safely
```

---

This architecture ensures:
✅ Security (Prepared statements, password hashing)
✅ Scalability (Proper database schema)
✅ Maintainability (Clean separation of concerns)
✅ User Experience (AJAX, clear messages)
✅ Performance (Indexed columns, efficient queries)
