## 📊 Database Schema

### Users Table
```
id, name, email, password (hashed), user_type, phone, address, city, state, 
profile_image, bio, is_verified, created_at, updated_at
```

### Services Table
```
id, name, description, category, icon
```

### Other Tables
- provider_services (Links providers to services)
- bookings (Service bookings)
- reviews (Customer reviews)

## 🔐 Login Flow

```
User Form → login.php → login_handler.php → Database Check → 
Session Created → Redirect to Dashboard
```

### Login Handler Logic:
1. Check for admin credentials (hardcoded)
2. Query database for user email
3. Verify password using password_verify()
4. Create session if valid
5. Return JSON response with redirect URL

## 🛠️ Troubleshooting

| Problem | Solution |
|---------|----------|
